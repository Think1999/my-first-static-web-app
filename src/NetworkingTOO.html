Network Topology

What is Network Topology? It is defined as a network's virtual shape or structure. Today, we will cover the standard Network Topologies. We will look at the shapes, how data is transmitted, and some of the strengths and weaknesses of each.


Star Topology

Pros:
• Used by home networks
• Central connection point called hub node
• Connect to the hub using Unshielded twisted pair (UTP)
• If a failure happens it will only take down one computer network
• Easy to add another computer to network

Cons:
• Requires more cable than the bus topology
• If the hub fails, the entire network will be lost.
• The cost is much higher, especially when using a switch or router


Bus Topology

In local area networks where bus topology is used, each node is connected to a single cable, by the help of interface connectors. This central cable is the backbone of the network and is known as the bus (thus the name). A signal from the source travels in both directions to all machines connected on the bus cable until it finds the intended recipient. If the machine address does not match the intended address for the data, the machine ignores the data. Alternatively, if the data matches the machine address, the data is accepted.
Fiber optic bus lines are a modern example of bus technology

Pros: inexpensive to implement
Cons: high cost of managing the network, single point of failure, data being transferred may be accessed by any node.


Ring topology

Is a type of network where all devices work as a ring and passes info from and to each device it is built to take on heavier load of data and is more efficient than Bus. Ring typology’s efficiency all depends on the devices in the network but it still has the ability to handle heavy network communications. The downside to ring typology is that one failure to transmit data can impact the entire network to make sure this does not happen you would use a dual ring structure where the information would be going counter clockwise and clockwise.


Mesh Topology

Networks that are interconnected with one another. It is use mainly for wireless networks. The topology is divided into two set-ups.
Full Mesh Topology and Partially Connected Mesh Topology.
Full Mesh Topology – Every device of the network is connected to each of the other devices in that network. Partially Connected Mesh Topology – At least two devices are connected to multiple other devices in that network. An inexpensive method to implement redundancy in the network.

Pros:
• Can handle multiple amount of data simultaneous which prevents a high amount of traffic.
• If one of the devices go down, the whole network will not break down as well.
• Integrating a new device will not interrupt the transmission of data between existing devices.

Cons:
• The cost for a mesh network is higher than other topologies
• The building and maintaining of the topology is daunting and time consuming
• Redundant connections could come into place creating a higher price and a possibility of reduced efficiency


Tree Network

A tree network looks like how it sounds. There is a central hub which would then be branched out with star topologies

Pros:
• It is the best topology for a large computer network, for which a star or ring topology are unsuitable due to the sheer scale of the entire network. Tree topology divides the whole network into parts, that are easily manageable.
• The topology makes it possible to have a point-to-point network.
• All computers have access to their immediate neighbors in the network and also the central hub. This kind of network makes it possible for multiple network devices to be connected with the central hub.
• It overcomes the limitation of star network topology, which has a limitation of hub connection points and the broadcast traffic induced limitation of a bus network topology.
• A tree network provides enough room for future expansion.

Cons:
• Dependence of the entire network on one central hub is a point of vulnerability for this topology. A failure of the central hub or failure of the main data trunk cable, can cripple the whole network.
• With increase in size beyond a point, the management becomes difficult.
